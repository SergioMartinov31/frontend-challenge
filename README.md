# Cats App

Frontend-приложение для просмотра котиков с бесконечной прокруткой,
возможностью добавления в избранное и сохранением состояния на клиенте.

Тестовое задание для стажёрской позиции Frontend Developer (VK, Учи.ру).

## Важно

API (https://thecatapi.com), используемое в проекте, может быть недоступно в некоторых регионах.  
Если изображения не загружаются, попробуйте открыть приложение с включённым VPN.  

В остальном приложение полностью работоспособно и соответствует требованиям задания.

## Функциональность

- по умолчанию открывается вкладка со всеми котиками
- бесконечная прокрутка списка котиков
- добавление и удаление котиков из избранного
- отдельная страница с избранными котиками
- сохранение избранных котиков после перезагрузки страницы
- sticky header для удобной навигации при длинной ленте
- hover и active-состояния карточек и кнопки лайка по макету
- на touch-устройствах кнопка лайка отображается сразу, так как hover-состояние на телефонах недоступно

## Архитектура

Проект организован по `Feature-Sliced Design`.

Используются основные слои:

- `app` - глобальные провайдеры, роутинг, store, layout
- `pages` - страницы приложения
- `widgets` - крупные самостоятельные UI-блоки, например header
- `features` - пользовательские сценарии, например добавление кота в избранное
- `entities` - сущности приложения, в данном случае `cat`

Такое разделение позволяет держать код модульным, изолировать бизнес-логику по зонам ответственности и упрощает дальнейшее масштабирование приложения.

## Архитектурные решения

- Серверные и клиентские данные разделены.
  Список котиков загружается через `RTK Query`, а избранные котики хранятся отдельно в Redux state.

- Для избранного хранится не только `id`, а целиком объект кота.
  Это позволяет корректно отображать страницу избранного после обновления страницы, не зависеть от уже загруженного списка и не делать дополнительные запросы.

- Избранные котики сохраняются в `localStorage`.
  Благодаря этому состояние переживает перезагрузку страницы и соответствует требованию ТЗ о хранении данных на клиенте.

- Бесконечная прокрутка реализована через `IntersectionObserver`.
  При появлении нижнего sentinel-элемента подгружается следующая страница котиков.

- Для мобильных устройств отдельно учтено отсутствие hover.
  На touch-экранах кнопка лайка всегда видна, чтобы взаимодействие оставалось очевидным и удобным.

- Для GitHub Pages используется `HashRouter`.
  Это решает проблему с прямым открытием внутренних роутов вроде `/favorites` на статическом хостинге.

## API

Приложение использует [The Cat API](https://thecatapi.com/).

Для корректной пагинации и получения не только случайных изображений нужен API key. Он передается через переменную окружения `VITE_CAT_API_KEY`.

## Запуск проекта

### 1. Клонировать репозиторий

```bash
git clone https://github.com/SergioMartinov31/frontend-challenge.git
cd frontend-challenge/cats-app
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Создать `.env`

В директории `cats-app` нужно создать файл `.env`:

```env
VITE_CAT_API_KEY=your_api_key
```

Получить API key можно на [The Cat API](https://thecatapi.com/).

### 4. Запустить проект

```bash
npm run dev
```

Приложение будет доступно по адресу:

```bash
http://localhost:5173
```

## Деплой

Проект деплоится на `GitHub Pages` через `GitHub Actions`.

Деплой автоматизирован: при пуше в ветку `master` workflow сам устанавливает зависимости, собирает проект и публикует содержимое `dist`.

Для сборки на GitHub нужно добавить repository secret:

```bash
VITE_CAT_API_KEY
```

После этого workflow сам соберет проект и опубликует содержимое `dist`.
